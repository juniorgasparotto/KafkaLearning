Publish/Subscribe é outro modelo em que uma mensagem (ou publicação, como é comumente chamada) é enviada a vários consumidores (ou assinantes) por meio de um tópico. O tópico é o link entre o publicador e assinante. As publicações serão recebidas por todos os assinantes. Quaisquer novos assinantes no mesmo tópico não receberão a publicação.

Nesse cenário, temos duas aplicações distintas (App1 e App2) que simulam duas APIs de diferentes propósitos que "escutam" o mesmo tópico (Chat).

Cada uma das apps tem seu próprio grupo de consumo (g1 e g2), isso faz com que cada mensagem produzida seja enviada para as duas apps ao mesmo tempo.
